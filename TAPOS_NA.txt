ANO ANG PROBLEMA:
═════════════════
Kahit may is_priority_encoded column na sa database at gumagana na ang
API filtering, lumalabas pa rin ang 637 non-priority projects sa Priority
Projects page. Dapat 0 lang kasi walang existing priority-encoded projects.

NAKITA KONG ROOT CAUSE:
════════════════════════
Ang problema ay TIMING ISSUE sa JavaScript variable assignment.

Yung ProjectsPage.type variable ay na-aassign HABANG NILO-LOAD PA LANG
ang JavaScript file - bago pa ma-set ang window.PROJECT_TYPE sa HTML.

Parang ganito ang nangyari:
  1. Browser: "O sige, i-load ko na projects.js"
  2. JavaScript: "Gawa ako ng ProjectsPage object..."
  3. JavaScript: "type = window.PROJECT_TYPE || 'all'"
  4. JavaScript: "Hmmm, wala pang window.PROJECT_TYPE... 'all' na lang"
  5. HTML: *Ngayon pa lang na-set: window.PROJECT_TYPE = 'priority'*
  6. JavaScript: "Huli ka na, naka-'all' na ako!" ❌

ANG SOLUSYON:
═════════════
HUWAG MAG-ASSIGN NG TYPE SA OBJECT CREATION!
Mag-assign sa loob ng init() method para ma-read na ang window.PROJECT_TYPE.

FILE NA BINAGO: static/js/projects.js
───────────────────────────────────────────────────────────────────────────
BEFORE (Line 11-13):
```javascript
const ProjectsPage = {
    type: window.PROJECT_TYPE || 'all',  // ❌ Masyadong maaga!
    currentPage: 1,
```

AFTER (Line 11-14):
```javascript
const ProjectsPage = {
    type: null,  // ✓ Wag muna, assign later
    currentPage: 1,
    pageSize: 50,
```

AT SA INIT() METHOD (Line 19-21):
```javascript
async init() {
    // ✓ DITO NA MAG-ASSIGN - after ma-set na ang window.PROJECT_TYPE
    this.type = window.PROJECT_TYPE || 'all';
    console.log('[PROJECTS] Initialized with type:', this.type);
```

  1. Browser nag-load ng projects.js ✓
