PRIORITY VS NON-PRIORITY PROJECTS FILTERING - FINAL FIX
============================================================

PROBLEMA:
---------
Lumalabas pa rin ang 637 Non-Priority projects sa Priority Projects page
kahit may is_priority_encoded field na sa database at naka-implement na
ang filtering sa API.

ROOT CAUSE IDENTIFIED:
----------------------
Sa static/js/projects.js, ang assignment ng type variable ay nangyayari
HABANG NILO-LOAD PA LANG ang JavaScript file, BAGO pa ma-set ang
window.PROJECT_TYPE sa HTML.

Sequence ng events BAGO ang fix:
1. Browser nag-load ng projects.js
2. JavaScript parser gumawa ng ProjectsPage object
3. Sa line 13: type: window.PROJECT_TYPE || 'all'
   - window.PROJECT_TYPE = undefined pa (hindi pa na-set)
   - Kaya naging type = 'all'
4. PAGKATAPOS, nag-load ng HTML inline script
5. Doon pa lang na-set: window.PROJECT_TYPE = 'priority'
6. Pero huli na - type variable ay 'all' na

SOLUTION IMPLEMENTED:
---------------------
FILE: static/js/projects.js

CHANGES:
Bago (Line 11-13):
```javascript
const ProjectsPage = {
    type: window.PROJECT_TYPE || 'all',
    currentPage: 1,
```

Pagkatapos (Line 11-14):
```javascript
const ProjectsPage = {
    type: null, // Will be set in init()
    currentPage: 1,
    pageSize: 50,
```

At sa init() method (Line 19-21):
```javascript
async init() {
    // Set type from window.PROJECT_TYPE (must be set AFTER the script that defines window.PROJECT_TYPE)
    this.type = window.PROJECT_TYPE || 'all';
    console.log('[PROJECTS] Initialized with type:', this.type);
```

BAKIT GUMAGANA ANG FIX:
------------------------
Ngayon, ang sequence ay:
1. Browser nag-load ng projects.js
2. JavaScript parser gumawa ng ProjectsPage object
3. type = null initially
4. Nag-load ng HTML inline script
5. Na-set: window.PROJECT_TYPE = 'priority'
6. DOMContentLoaded event fires
7. ProjectsPage.init() na-call
8. Dito na nag-assign: this.type = window.PROJECT_TYPE || 'all'
9. this.type = 'priority' ✓ TAMA NA!
10. API call: /api/v1/projects?type=priority ✓ TAMA!
11. Returns 0 projects (kasi walang pa is_priority_encoded='yes')
